Implement the complete booking lifecycle for Snow Connects with INSTANT BOOKING model. Customer pays and booking is immediately confirmed — no instructor approval required. Instructors have cancellation rights with rules.

═══════════════════════════════════════
PHASE 1: BOOKING & PAYMENT (Customer)
═══════════════════════════════════════

1. Customer completes booking flow (date → slots → participants → student info → summary)
2. On "Onayla ve Öde" tap:
   - System performs FINAL availability check on instructor's calendar
   - If slots are no longer available: show error "Bu saatler az önce başkası tarafından alındı. Lütfen başka saat seçin." and return to calendar
   - If available: continue to payment
3. Payment is captured immediately (full charge, not pre-auth)
   - For test mode: skip payment, mark as "paid_test"
4. Booking status: "confirmed" (instant)
5. Instructor's calendar slots are blocked
6. Customer sees confirmation screen: "🎿 Rezervasyonun onaylandı!"
7. Push notification to instructor: "Yeni rezervasyonun var! [Müşteri Adı], [Tarih] [Saat]'de [Kişi sayısı] kişi"

═══════════════════════════════════════
PHASE 2: INSTRUCTOR CANCELLATION RIGHTS
═══════════════════════════════════════

Instructors do NOT have approve/decline option. Booking is automatically confirmed.
However, instructors CAN cancel with reason in case of emergency.

When instructor taps "Rezervasyonu İptal Et":
- Must select a reason:
  * "Hastalık/yaralanma"
  * "Acil durum"
  * "Hava şartları uygunsuz"
  * "Diğer" (text input required)
- Status changes to "cancelled_by_instructor"
- Customer gets 100% refund
- Customer gets 10% discount coupon for next booking (apology)
- Customer push notification: "Maalesef Hasan Hoca rezervasyonunu iptal etmek zorunda kaldı. Tam iaden yapıldı ve sana özel %10 indirim kuponu hediyemiz var. Aynı tarihte müsait diğer eğitmenleri görmek ister misin?"
- Show alternative instructors at same resort and date range
- Increment instructor's cancel counter:
  * If cancelled 24+ hours before lesson: counter +1
  * If cancelled less than 24 hours before: counter +2 (penalty)

═══════════════════════════════════════
PHASE 3: INSTRUCTOR DISCIPLINE SYSTEM
═══════════════════════════════════════

Calculate instructor's cancellation rate over last 20 bookings:

- 0-5% cancellation rate: 
  * "Güvenilir Eğitmen" badge displayed on profile
  * Boosted in search rankings
- 5-10% cancellation rate:
  * Show warning on instructor's own dashboard
  * No public badge
- 10%+ cancellation rate:
  * Account flagged for admin review
  * Show banner: "Hesabınız incelemede. İptal oranınızı düşürün."
  * Reduced ranking in search results
- 20%+ cancellation rate:
  * Account auto-suspended pending admin decision
  * Cannot receive new bookings until reviewed

Force majeure exceptions (do NOT count toward cancel rate):
- Resort closed due to weather (system-detected)
- Admin manually marks as "force_majeure"

Display on instructor profile (visible to customers):
- "Yanıt oranı: %98"
- "İptal oranı: %2"  
- "Tamamlanan ders: 156"
- "Ortalama puan: 4.9 ⭐"

═══════════════════════════════════════
PHASE 4: CUSTOMER CANCELLATION RULES
═══════════════════════════════════════

Calculate hours between cancellation request and lesson start time:

- More than 48 hours: 100% refund
- 24-48 hours: 50% refund (50% paid to instructor as compensation)
- Less than 24 hours: NO refund (full amount paid to instructor)

Customer cancellation flow:
- Customer taps "Rezervasyonu İptal Et" in booking details
- Show clear refund preview based on time remaining:
  * "Şu an iptal edersen: ₺X.XXX iade alacaksın"
- Confirmation dialog
- On confirm: status → "cancelled_by_customer", refund processed

═══════════════════════════════════════
PHASE 5: PRE-LESSON
═══════════════════════════════════════

After confirmation:
- Both parties can message each other
- 24 hours before: push notification to both ("Yarın dersin var!")
- 2 hours before: push notification to both ("Dersine 2 saat kaldı")
- 30 minutes before: "Derse Başla" button activates for instructor

═══════════════════════════════════════
PHASE 6: LESSON DAY
═══════════════════════════════════════

1. Instructor taps "Derse Başladım"
   - Captures GPS location + timestamp
   - Customer notification: "Hasan Hoca derse başladı"
   - Status: "instructor_started"
2. Customer taps "Eğitmenimle Buluştum"
   - Captures timestamp
   - Status: "in_progress"
3. After lesson end time + 10 minutes, instructor sees "Dersi Tamamladım" button
4. Instructor taps "Dersi Tamamladım"
   - Captures timestamp
   - Status: "completion_pending"

═══════════════════════════════════════
PHASE 7: POST-LESSON CONFIRMATION
═══════════════════════════════════════

1. Customer push notification: "Dersin nasıldı? Hasan Hoca'yı değerlendir"
2. Customer opens review screen with two options:
   
   OPTION A — "Onayla ve Değerlendir":
   - 1-5 star rating (required)
   - Comment (optional, 500 char max)
   - "Onayla ve Tamamla" button
   - On submit:
     * Status: "completed"
     * Review saved to instructor profile
     * Payout countdown starts (21 business days from completion)
   
   OPTION B — "Bir Sorun Bildir":
   - Reason selection:
     * "Eğitmen gelmedi"
     * "Ders eksik kaldı/erken bitti"
     * "Eğitmen profesyonel davranmadı"
     * "Diğer" (text)
   - Optional photo upload
   - Status: "disputed"
   - Admin notification triggered
   - Payout paused until resolution

3. If customer doesn't respond within 24 hours of "completion_pending":
   - Auto-complete: status → "completed"
   - No review submitted
   - Payout countdown starts

═══════════════════════════════════════
PHASE 8: DISPUTE RESOLUTION (Admin)
═══════════════════════════════════════

When booking status is "disputed":
- Admin sees in "İtirazlar" panel
- Admin can view:
  * All booking details
  * Both parties' message history
  * GPS location + timestamps from instructor "started" action
  * Customer's reported issue + photos
- Both parties can submit additional evidence
- Admin actions:
  * "Müşteri haklı" → full refund, payout cancelled, instructor warning
  * "Eğitmen haklı" → no refund, payout proceeds
  * "Kısmi çözüm" → custom refund amount, partial payout

═══════════════════════════════════════
PHASE 9: WEATHER & FORCE MAJEURE
═══════════════════════════════════════

Add resort status field:
- 'open' (normal operations)
- 'partial_closure' (some lifts closed)
- 'closed' (full closure due to weather/safety)

When admin marks resort as "closed":
- All confirmed bookings for that resort on that date:
  * Auto-cancelled with reason "force_majeure"
  * Customer gets 100% refund
  * Instructor's cancel counter does NOT increment
  * Customer can rebook for different date with no fee
